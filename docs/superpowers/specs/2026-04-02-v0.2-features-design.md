## 1. Seeded Random

### Problem

`Math.random()` makes output non-reproducible. Tests are flaky, seed data changes between runs.

### Design

Add `seed?: number` to `TGenerateOptions`. When provided, use a seeded PRNG instead of `Math.random()`.

**PRNG algorithm:** mulberry32 — 32-bit, fast, ~10 lines, sufficient quality for name selection.

**Internal change:** `pickRandom` and `pickWeighted` accept `rng: () => number` parameter instead of calling `Math.random()` directly.

```typescript
// Public API
generate({ seed: 42 })  // always returns the same name
generate({ seed: 42, gender: EGender.Female })  // deterministic + filter
generateMany(10, { seed: 42 })  // same 10 names every time
```

**Backward compatible:** no seed → `Math.random()` as before.

### Files

- `src/random.ts` — add `mulberry32(seed)`, refactor `pickRandom`/`pickWeighted` to accept `rng`
- `src/types.ts` — add `seed?: number` to `TGenerateOptions`
- `src/generator.ts` — pass `rng` through to pick functions

## 2. Romanization

### Problem

Vietnamese diacritics are incompatible with URLs, usernames, email addresses, and ASCII-only systems.

### Design

Public standalone function + integrated into `INameResult`.

**Algorithm:** `String.normalize('NFD')` + strip combining marks via regex. Special-case `Đ/đ` → `D/d` (does not decompose via NFD).

```typescript
// Standalone — public API
romanize('Nguyễn Văn An')  // → 'Nguyen Van An'

// In result
const result = generate();
result.romanized.surname   // 'Nguyen'
result.romanized.fullName  // 'Nguyen Van An'
```

### Types

```typescript
interface IRomanizedName {
  surname: string;
  middleName: string;
  givenName: string;
  fullName: string;
}
```

`INameResult` adds field: `romanized: IRomanizedName`

### Files

- `src/romanize.ts` — `romanize(input: string): string` function
- `src/types.ts` — add `IRomanizedName`, add `romanized` to `INameResult`
- `src/generator.ts` — compute `romanized` from generated parts
- `src/index.ts` — export `romanize`

## 3. Output Format Options

### Problem

Users need names in different formats (abbreviated, Western order, URL slug) and currently must transform manually.

### Design

New enum `ENameFormat` with 4 values. Option accepts single or array. Result always returns `formatted` as a record.

```typescript
enum ENameFormat {
  Full = 'full',            // "Nguyễn Văn An" (default)
  Abbreviated = 'abbreviated', // "N.V. An"
  Reversed = 'reversed',    // "An Nguyễn Văn" (Western order)
  Slug = 'slug',            // "nguyen-van-an" (romanized + kebab-case)
}
```

### Options

```typescript
// TGenerateOptions adds:
format?: ENameFormat | ENameFormat[];
```

### Result

```typescript
// INameResult adds:
formatted: Partial<Record<ENameFormat, string>>;
```

- No format specified → `formatted` contains `{ full: '...' }`
- Single format → `formatted` contains that format
- Array → `formatted` contains all requested formats
- `fullName` field unchanged for backward compatibility

### Format logic

| Format | Rule | Example |
|--------|------|---------|
| `full` | `{surname} {middleName} {givenName}` | Nguyễn Văn An |
| `abbreviated` | `{S}.{M}. {givenName}` (first char of surname/middle, keep full given) | N.V. An |
| `reversed` | `{givenName} {surname} {middleName}` | An Nguyễn Văn |
| `slug` | `romanize(full)` → lowercase → replace spaces with `-` | nguyen-van-an |

**Edge cases:**
- No middle name: `abbreviated` → `N. An`, `reversed` → `An Nguyễn`, `slug` omits empty segment
- Compound surname (Tôn Thất): `abbreviated` takes first char of each word → `T.T. An`
- Compound given name (Bảo Long): `abbreviated` keeps full given → `N.V. Bảo Long`, `reversed` → `Bảo Long Nguyễn Văn`

### Files

- `src/format.ts` — `formatName(result: INameResult, format: ENameFormat): string`
- `src/types.ts` — add `ENameFormat`, update `TGenerateOptions`, update `INameResult`
- `src/generator.ts` — compute `formatted` field
- `src/index.ts` — export `ENameFormat`, `formatName`

## 4. CLI

### Problem

No way to use the library without writing code. CLI enables quick use, demos, and scripting.

### Design

`npx vietnamese-name-generator` runs CLI. Zero additional dependencies — parse `process.argv` manually.

### Usage

```bash
# Basic
vietnamese-name-generator                          # 1 random name
vietnamese-name-generator --count 10               # 10 names
vietnamese-name-generator --gender female --region south --era modern

# New features
vietnamese-name-generator --seed 42                # deterministic
vietnamese-name-generator --format slug            # single format
vietnamese-name-generator --format full,slug       # multi format

# Output
vietnamese-name-generator --count 5                # table (default, human-readable)
vietnamese-name-generator --count 5 --json         # JSON (for piping/scripting)

# Help
vietnamese-name-generator --help
vietnamese-name-generator --version
```

### Flags

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--count, -n` | number | 1 | Number of names |
| `--gender, -g` | string | random | male, female, unisex |
| `--region, -r` | string | random | north, central, south |
| `--era, -e` | string | random | traditional, modern |
| `--seed, -s` | number | none | Seed for deterministic output |
| `--format, -f` | string | full | Comma-separated formats |
| `--compound` | boolean | undefined | Force compound names |
| `--meaning` | string | none | Meaning category filter |
| `--no-middle` | boolean | false | Omit middle name |
| `--json` | boolean | false | JSON output |
| `--help, -h` | boolean | - | Show help |
| `--version, -v` | boolean | - | Show version |

### Output formats

**Table (default):**
```
  Nguyễn Văn An      (male, north, modern)
  Trần Thị Mai       (female, north, traditional)
```

**JSON (`--json`):**
```json
[{"surname":"Nguyễn","middleName":"Văn","givenName":"An","fullName":"Nguyễn Văn An","gender":"male","region":"north","era":"modern","romanized":{"surname":"Nguyen","middleName":"Van","givenName":"An","fullName":"Nguyen Van An"},"formatted":{"full":"Nguyễn Văn An"}}]
```

### Build

- `tsup.config.ts`: add `src/cli.ts` as second entry point
- `package.json`: add `"bin": { "vietnamese-name-generator": "./dist/cli.cjs" }`
- `cli.ts` includes shebang `#!/usr/bin/env node`

### Files

- `src/cli.ts` — argv parsing, output formatting, help text
- `tsup.config.ts` — add CLI entry point
- `package.json` — add `bin` field, update `files` if needed

## Backward Compatibility

All changes are additive:
- `fullName` field unchanged
- No seed → same random behavior as v0.1.x
- No format → `formatted` defaults to `{ full: fullName }`
- Existing function signatures unchanged (all new params are optional)
- New exports: `romanize`, `formatName`, `ENameFormat`, `IRomanizedName`

## Dependencies

Zero new runtime dependencies. CLI arg parsing is manual (~50 lines).

## Test Plan

- **Seed:** same seed → same output across calls; different seed → different output; no seed → random
- **Romanize:** diacritics removal for all Vietnamese characters; `Đ/đ` handling; empty string; standalone usage
- **Format:** each format type; multi-format array; edge cases (no middle name, compound surname)
- **CLI:** integration tests via child_process spawn; flag parsing; table/JSON output; --help/--version
